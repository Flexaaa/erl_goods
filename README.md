Приложение на erlang, которое получает данные и складирует их в БД mnesia.
Для компиляции и запуска используется Makefile 
compile - компилирует основной код и библиотеки
compile-tests - собирает тесты
run-app - запускает приложение
tests - запускает тесты

Сборка - просто компиляция исходников в нужную папку (rebar не использовался)
Тесты не покрывают все кейсы, как помощь при настройке фильтров для работы с БД и запросами.

Проверял на локальном сервере с данными из https://dummyjson.com/products (просто без goodbiydpi не достал, на линуксе не настроено)

Да я знаю что решение не сильно скалируемое и наверняка через какое-то количество времени БД захлебнётся от количества данных.
Mnesia - решил посмотреть, явно с sql было бы лучше в плане хранения и запросов.
Но по условиям выдавать всё - это уже ограничение, параметры должны работать (id проверил - остальное долго запускать, можно тест добавить)

ЗНАЧЕНИЯ ПЕРЕМЕННЫХ

Все переменные находятся в приложении goods_app

| Имя | Значение по умолчанию | Описание |
| :---: | :--- | :--- |
| port | 5432 |  порт для получения данных, сохранённых в приложении |
| url | [https://dummyjson.com/products](https://dummyjson.com/products) | источник данных |
| timeout | 30000 (30 секунд) | таймаут ожидания данных от источника |
| module | goods_db | модуль для сохранения данных в БД |
| partial | false | флаг для включения получения данных по частям |
| chunk_size | 10 | количество данных в одной передаче (используется апи dummyjson.com с использованием запросов со сдвигом и лимитом) |

ПРОВЕРКА

Компилировал на Ubuntu 24 и проверял на erlang 25, но по идее библиотеки старые и никаких особо новых функций нет - должно и на более ранних версиях работать. Дополнительно проверено на windows 11 (с использование zapret).
